其余说明：

1.	请简单描述你们组在Beer Bench和Tips Bench两个环节分别的策略（描述你使用的主要方法以及软硬件工具）。

Beer Bench:
在openclaw中创建一个agent，用于服务需要点酒的顾客。使用提示词在openclaw的AGENTS.md中写好agent工作流的描述。和Openclaw对话，其会引导顾客进入“点酒-放酒-送酒-要小费”的流程。放酒时，提前录制好多种位置的放酒动作，届时openclaw使用让机械臂播放不同动作的skills放酒杯。送酒过程通过一个前往指定目标颜色的skill实现，同时增加了openclaw隔一段时间通过小车拍照skill来检测小车是否被障碍物阻挡，如果是，则openclaw控制小车避障。送达后，openclaw调用转圈skill转圈来吸引顾客注意进行庆祝。
Tips Bench:
创建另一个openclaw的agent，用于寻找可能点酒的顾客。其先通过摄像头查看视野中的顾客，然后挑选最有可能下单的顾客，引导其下单，如不行则放弃，成功则将下单请求通过cron定时命令短时间后转交给Beer Bench中的agent处理。
这个寻找潜在消费者的agent，使用cron设置为2分钟唤醒一次，以对应真实酒馆前台的工作情况。

软硬件工具：电子小车、机器臂、usb摄像头等智谱标配硬件。硬件均连接到windows下的ubuntu（使用WSL）。使用windsurf连接到WSL环境中进行编程，AI辅助编程使用codex插件。Openclaw使用GLM-5模型。

2.	你觉得你们组在本次比赛中表现的亮点是什么？如果可以的话，请根据你写的skills文档/其他脚本或代码回答

使用在线算法实现小车自行前往指定颜色的目标（封装成skill）；GLM-5模拟酒馆服务员时的情商很高！其对提示词的理解很到位！使用了openclaw的定时功能和多agent交互的理念。

3.	在本次比赛中，你觉得你们组的哪些尝试最有效？哪些是无效的？为什么？

有效：使用接入了GLM-5的openclaw控制硬件时，其自主调用skills的积极性很高。使用gpt编写小车根据颜色前往指定目标的函数，几乎一次就实现了。酒馆前台和后台两个agent成功交互。

无效：小车送酒到达后，返回出发地，由于对小车本身特性短时间掌握不清，按点回溯的返回程序效果差，后续了解到可以用小车自带的按点导航功能便捷实现。

4.	你在本次比赛中最大的收获是什么？这次比赛从什么程度和方面加深了你对openclaw的理解？

Openclaw的ReAct模式十分适合操控需要根据环境情况实时决策的机器人，我们实践了openclaw的设计模式在硬件上的应用，以及体验了GLM-5的强大智能水平。

5.	这次比赛中有没有你们组没来得及实现的功能，你对openclaw未来的应用有什么设想和展望？

带优化完成：对机械臂的控制颗粒度不够细，没有给机械臂增加视觉功能，openclaw的决策速度有待提升，Tips Bench环节可以增加更多有创意的ideas。
Openclaw的应用：使用反应速度快的模型，结合openclaw的强规划模式，实现机器人自主决策，像酒馆这样不是非常追求效率的场景完全可行，且编程非常容易。在软件方面，将更多平台的API暴露给Openclaw供其搭建生态是一个好的方向。
Openclaw的稳定性尚不完美，使得其在及时决策场景下表现有限，在长时间后台任务中表现更好，建议对及时决策场景做特定优化，比如简化ReAct模式，减少推理时间。
